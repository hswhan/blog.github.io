C++Primer(3)——const限定符和自定义类型

发表于 2019-09-17 | 分类于 C++Primer | 本文总阅读量 次
字数统计: 1.2k 字 | 阅读时长 ≈ 4 分钟

const限定符

当我们希望定义一个变量，它的值不能被改变(只读)时，可以用const限定符对变量类型加以限定。因为const对象一旦创建好后它的值就不能再改变(只能在const类型的对象上执行不改变其内容的操作)，所以const对象必须初始化。
默认状态下，const对象仅在文件内有效编译器在编译过程中把用到该变量的地方都替换成对应的值。如果要想在多个文件共享const对象，必须在变量定义之前添加extern关键字。

const的引用

可以把引用绑定到const对象上，称之为对常量的引用。
初始化常量引用时允许使用任意表达式作为初始值，只要表达式的结果能够转换成引用类型，允许一个常量引用绑定非常量的对象字面值，甚至是一个一般表达式。

指针和const

与引用一样，指针也可以指向常量或者非常量，指向常量的指针不能改变其所指对象的值。

const指针，指针是一个对象而引用不是，因此和其他对象类型一样可以办指针本身定义为常量，即常量指针且必须被初始化。一旦初始化完成了，它的值(也就是存放在指针中的地址)就不能再改变了。

通过从右往左阅读，离curErr最近的符号是const，这就意味着curErr本身就是一个常量对象，下一个声明符是*，所以意味着它是一个常量指针。

顶层const与底层const

顶层const表示指针本身是个常量，而底层const表示指针所指的对象是一个常量。更一般的可以表示任意对象是常量，比如算术类型、类、指针等，底层const则与指针和引用等复合类型有关。

constexpr和常量表达式

常量表达式是指不会改变并且在编译过程中就能得到计算结果的表达式。像get_size()这种需要在具体运行的时候调用这个函数在知道值的不是常量表达式。
constexpr变量，将变量声明为constexpr以便由编译器来验证变量的值是否为一个常量表达式。声明为constexpr的变量一定是一个常量，且必须用常量表达式来初始化。

算术类型、引用、指针都属于字面值类型。自定义类、IO库、string类型等不属于字面值类型。

处理类型

类型别名

类型别名是一个名字，它是某种类型的同义词。

auto类型说明符

auto让编译器通过初始值来推算变量的类型，一条声明只能有一个基本数据类型。

auto一般会忽略掉顶层const

decltype类型指示符

decltype它的作用是选择并返回操作数的类型。如果表达式的内容是解引用操作，则decltype将得到引用类型。
decltype((variable))的结果永远是引用，而decltype(variable)的结果只有当variable本身是一个引用时才是引用。

自定义数据结构

以struct关键字开始，紧跟着类名和类体，花括号结束后必须写一个分号表示声明符的结束。为了确保各个文件中的类的定义一致，类通常被定义在头文件中，且类所在的头文件应与类的名字一样。
