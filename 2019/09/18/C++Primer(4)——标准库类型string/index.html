string对象读取操作

在输入运算符”>>”执行读取操作的时候，string对象会自动忽略开头的空白(空格、换行、制表)。 并且用while循环和cin读取未知数量的string对象时，会以空白符作为单词分隔，而getline读取一整行可以保留输入时的空白符，触发返回的换行符会被丢弃，得到的string中并不包含换行符。

size()函数

size函数返回一个int值或者unsigned值都是合情合理的，但实质上size函数得返回值是一个string::size_type类型的值。
如果一个表达式中已经有了size()函数了就不要再使用int，防止混用int和unsigned带来问题。

string对象大小比较方面

1.如果两个string对象长度不同，而且较短的string对象的每个字符和较长的string对象对应位置的字符相同，则较短的string小于较长的string对象。
2.如果两个string对象在某些对应的位置不一样，则string对象比较的是第一对相异字符比较的结果。

string对象和字面值相加

必须确保每个加法运算符的两侧运算对象至少有一个是string。
处理string对象中的具体某一个字符，在cctype的头文件中定义了一组标准库函数来处理这部分工作，包括判断字符是字母、数字、控制符、小写等等，具体见C++Primer P.82。

除了通过for语句控制范围，还可以用下标，或者叫索引，string对象的下标[]必须大于0小于s.size()。使用超出此范围的下标会发生不可预知的结果。
