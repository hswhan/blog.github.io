寻找两个有序数组的中位数——Leetcode(2)

发表于 2019-09-16 | 分类于 Leetcode | 本文总阅读量 次
字数统计: 702 字 | 阅读时长 ≈ 3 分钟

题目

给定两个大小为 m 和 n 的有序数组 nums1 和 nums2。请你找出这两个有序数组的中位数，并且要求算法的时间复杂度为O(log(m + n))。你可以假设 nums1 和 nums2 不会同时为空。

解法

第一次做没看清对时间复杂度的要求，上来直接合并数组然后再取中位数了方法太蠢，不列了，还想着这题咋这简单。关于时间复杂度，另开一篇说一下了。可以看出，蠢方法的时间复杂度是O(m+n)的，想要达到O(log)对数阶的话，只有使用二分查找法了。(注意，给的是两个有序的数组)

二分法

设两个数组A, B的长度分别为l1, l2。
当l1+l2为奇数，k=(l1+l2)//2。
当l1+l2为偶数，分别找到第(l1+l2)/2-1和第(l1+l2)/2个数求均值。
二分法，每次将两个数组分成两半，中位数分别为m1和n1,中位数的下标分别为i1和i2。
1.若i1+i2< k，即第k个数一定不在某个数组的前半部分，当m1>n1时，第k个数一定不在B的前半部分，下一次搜索改为在A和B的后半部分里寻找第k-i2-1个数；当m1<=m2时,同理，问题变为在A的后半部分和B里寻找第k-i1-1个数。
2.若i1+i2>=k，即第k个数一定不在某个数组的后半部分，当m1>m2时，第k个数一定不在A的后半部分，问题变为，在A的前半部分和B中寻找第k个数；反之不在B的后半部分，问题也相应改变，这样搜索的范围就减小了。
递归函数的终止条件：
1、当A（B）为空，直接返回B[k-1]（A[k-1]）；
2、当k=1时，返回min(A[0], B[0])；
